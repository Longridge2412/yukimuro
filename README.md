# ❄️ 雪室出入庫管理 v2.0

雪室の入庫・出庫をタブレットで記録し、Googleスプレッドシートで自動管理するWebアプリです。
v2.0ではロット管理・FIFO出庫・保管日数ベースの清算計算に対応しました。

## 概要

- **利用者**（米農家・酒造・公社など）がタブレットで入庫・出庫を記録
- **管理者**がGoogleスプレッドシートで在庫確認・清算管理
- 入庫ごとにロットIDが自動採番され、出庫はFIFO（先入先出）で処理
- 保管日数 × 数量 × 単価で清算額を自動計算
- パスワード認証でアクセスを制限

## v2.0の主な変更点

- **ロット管理**: 入庫ごとにロットIDを自動採番（例: HST-R7-A）し、個別に追跡
- **FIFO出庫**: 古いロットから優先的に出庫。保管日数を自動計算
- **清算自動計算**: 出庫時に「保管日数 × 数量 × 単価」で清算額を算出
- **単位の整理**: 30kg・フレコン・個・本・箱・ケース・袋の7種類に統一（10kg・5kg・俵・kgを廃止）
- **「作業者」→「利用者」**: 誰の荷物かを明確に記録する概念に変更
- **管理者モード拡張**: 在庫（ロットベース）・履歴・清算・設定の4タブ構成

## セットアップ手順

### 1. Googleスプレッドシートを作成

1. [Google スプレッドシート](https://sheets.google.com) で新規スプレッドシートを作成
2. メニューの **拡張機能 → Apps Script** を開く
3. `gas/Code.gs` の内容をすべてコピーして貼り付け
4. **保存**（Ctrl+S）
5. `setupSpreadsheet` 関数を実行（▶ ボタン）→ 9つのシートが自動作成されます
6. 初回は権限の承認を求められるので「許可」してください

### 2. APIキーを設定（任意）

不正アクセスを防ぐため、APIキーの設定を推奨します:

1. `Code.gs` の `API_KEY` 変数にキーを入力（例: `const API_KEY = "mySecretKey123";`）
2. アプリの管理者設定でも同じキーを入力します（後述）

### 3. マスタデータを入力

スプレッドシートの各シートにデータを入力します:

- **利用者マスタ**: 利用者の名前とコードを入力（No.・利用者名・コードが必須）。コードはロットID生成に使われます（例: コード「HST」→ ロットID「HST-R7-A」）
- **雪室一覧**: 雪室の名前を入力（No. と 雪室名 が必須）
- **品目マスタ**: 米の品種や日本酒の銘柄を入力（プリセット9品種が入っています。自由に追加可能）

### 4. 保管単価を設定（重要）

**単価マスタ**シートで、単位ごとの保管単価を管理者が設定します。この設定により、出庫時の清算額が自動計算されます。

| 単位 | 単価（円/個/日） | 備考 |
|------|-----------------|------|
| 30kg | 4 | 1袋1日あたり4円 |
| フレコン | 4 | 必要に応じて変更可能 |
| 個 | 4 | 必要に応じて変更可能 |
| 本 | 4 | 必要に応じて変更可能 |
| 箱 | 4 | 必要に応じて変更可能 |
| ケース | 4 | 必要に応じて変更可能 |
| 袋 | 4 | 必要に応じて変更可能 |

初期値はすべて4円/個/日ですが、フレコンなど大型の保管物は単価を高くするなど、管理者が自由に変更できます。単価の変更はスプレッドシートの単価マスタシートを直接編集してください。変更は次回の出庫・清算から反映されます。

### 5. GASをWebアプリとしてデプロイ

1. Apps Script エディタで **デプロイ → 新しいデプロイ**
2. 種類: **ウェブアプリ**
3. 設定:
   - 説明: 「雪室出入庫管理API v2.0」
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
4. **デプロイ** をクリック
5. 表示される **Web アプリの URL** をコピー

### 6. アプリを公開

GitHub Pages などで `index.html` をホスティングするか、タブレットのブラウザで直接開きます。

### 7. アプリにGAS URLを設定

1. アプリをブラウザで開く
2. パスワード `yukimuro2026` を入力してログイン
3. ヘッダー部分を **3秒長押し** → 管理者モードを表示
4. **⚙️ 設定 → 🔗 GAS接続設定** を開く
5. GAS WebアプリURLを貼り付け
6. APIキーを設定した場合はそちらも入力
7. **接続テスト** で動作確認 → **保存して反映**

## 使い方

### 利用者（タブレット）

**入庫の流れ:**

1. ブラウザでアプリを開き、パスワードを入力してログイン
2. **入庫** をタップ
3. 利用者 → 雪室 → 品目 → 数量・単位 を順に選択
4. 確認画面でロットID（自動採番）を確認し、**記録する** をタップ
5. スプレッドシートのロット台帳・入出庫記録に自動で書き込まれます

**出庫の流れ:**

1. **出庫** をタップ
2. 利用者 → 雪室 を選択
3. 該当する保管中ロットがFIFO順（入庫日が古い順）に表示されます
4. 出庫するロットを選択し、出庫数を入力
5. 清算プレビュー（保管日数 × 数量 × 単価）を確認し、**記録する** をタップ
6. ロット台帳の残数が更新され、清算明細に清算額が記録されます

### 管理者

ヘッダー部分を **3秒長押し** で管理者モードを表示します。4つのタブがあります:

- **📊 在庫**: ロットベースの在庫一覧。利用者・雪室ごとにグループ化して表示
- **📋 履歴**: 入出庫の全記録。CSV/JSONでエクスポート可能
- **💰 清算**: 利用者ごとの清算明細（保管日数・数量・単価・清算額）を一覧表示
- **⚙️ 設定**: GAS接続設定、利用者・雪室・品目のマスタ管理、ログアウト

## 清算の仕組み

清算額は以下の計算式で自動算出されます:

```
清算額 = 出庫数 × 保管日数 × 単価（円/個/日）
```

- **保管日数**: 入庫日から出庫日までの日数（自動計算）
- **単価**: 単価マスタシートで単位ごとに設定（管理者が変更可能）
- **FIFO**: 同じ利用者・雪室・品目に複数ロットがある場合、入庫日が古いものから優先的に出庫されます

年度末（3月31日）に未出庫のロットがある場合は、年次清算処理を実行して当年度分の清算を確定できます。

## スプレッドシート構成（9シート）

| シート名 | 内容 |
|---------|------|
| 利用者マスタ | 利用者の名前・コード（→アプリの選択肢・ロットID生成に使用） |
| 雪室一覧 | 雪室名・場所（→アプリの選択肢に反映） |
| 品目マスタ | 米の品種・日本酒の銘柄（→アプリの選択肢に反映） |
| 単価マスタ | 単位ごとの保管単価（円/個/日）。管理者が設定・変更可能 |
| ロット台帳 | 入庫ごとのロット情報（ロットID・入庫日・数量・残数・ステータス） |
| 入出庫記録 | アプリから自動書き込み（入庫・出庫の全記録にロットID付き） |
| 清算明細 | 出庫時の清算記録（保管日数・数量・単価・清算額） |
| 作業記録 | 施設管理の作業ログ（将来拡張用） |

## オフライン対応

GAS未接続時（ネットワーク障害など）でもローカルに記録が保存されます。接続復帰後に管理者モードからCSV/JSONでエクスポートできます。

## セキュリティ

- **パスワード認証**: アプリ起動時にパスワード入力が必要
- **APIキー**: GAS側で設定可能。アプリとGAS間の通信を保護
- **GAS URL非公開**: URLはコードに含まれず、管理者が端末ごとに設定（localStorageに保存）

## 技術スタック

- Google Apps Script（Webアプリとしてデプロイ）
- Google スプレッドシート（データベース代わり）

## GASコード更新時の注意

GASのコードを更新した場合は、Apps Scriptエディタで **デプロイ → デプロイを管理 → 編集（鉛筆アイコン）→ バージョンを「新しいバージョン」に変更 → デプロイ** を実行してください。URLは変わりませんが、新しいバージョンをデプロイしないと変更が反映されません。
